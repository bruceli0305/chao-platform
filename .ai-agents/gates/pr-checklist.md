# Gate: pr-checklist

## 1. 目的

作为工程门禁，提供可验证证据。

## 2. 适用

根据任务等级和风险类型启用。

## 3. 输出模板

```md
# Gate Result: pr-checklist

## 任务编号
## 命令 / 检查方式
## 执行环境
## 结果
通过 / 失败 / 未执行
## 输出摘要
## 未覆盖内容
## 残余风险
```

## 4. 失败处理

失败不得交付，必须回流到对应角色处理。

## 数据边界 Checklist

```text
- [ ] 已标明新增 / 修改的数据类型；
- [ ] 已标明数据分级；
- [ ] 已标明主存储位置；
- [ ] 未把 Secret / 生产数据 / 隐私写入 PostgreSQL、Markdown、日志或向量库；
- [ ] 如新增向量索引来源，已完成脱敏和白名单审查；
- [ ] 如新增 artifact，已标明保留期限和访问权限；
- [ ] 如新增环境变量，已更新 .env.example 且未提交真实值。
```
