# 任务等级规则 v3

## L1

低风险单点任务，必须同时满足：

```text
单点修改或少量文件修改；
不改 API；
不改数据；
不改权限；
不改部署；
不增依赖；
不涉及 Secret；
不改架构；
一种轻量验证可闭环。
```

## L2

普通功能或普通联动任务，满足任一即最低 L2：

```text
新增或修改普通页面；
新增或修改普通接口；
前后端联动；
多个文件协作；
普通 DTO / 字段 / 状态变化；
需要测试或文档同步。
```

## L3

复杂或高风险任务，满足任一即最低 L3：

```text
数据库结构变化；
数据迁移；
权限模型变化；
认证 / 鉴权变化；
生产配置变化；
部署方式变化；
新增第三方服务；
成本明显增加；
新技术栈；
架构边界变化；
Secret 相关；
无法简单回滚。
```

## L4

战略级工程，不直接执行：

```text
多仓库协同；
平台级架构升级；
多系统迁移；
长期智能体体系建设；
CI/CD、权限、数据、部署整体重构。
```

## 数据边界定级规则

```text
新增普通文档：L1 / L2；
新增任务状态字段：最低 L2；
新增 PostgreSQL 表：最低 L3；
新增向量索引来源：最低 L3；
新增 Secret 或环境变量：最低 L3；
涉及生产数据：最低 L3，默认阻塞，需专项确认；
Secret 泄露：高风险阻断，进入事故流程。
```
