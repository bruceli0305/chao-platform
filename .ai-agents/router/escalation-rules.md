# 自动升级规则 v3

| 当前情况 | 升级结果 |
|---|---|
| L1 发现 API 契约变化 | 升级 L2 |
| L1 发现多个模块联动 | 升级 L2 |
| L1 发现新增依赖 | 升级 L2，启用户部 |
| L1 / L2 发现数据库结构变化 | 升级 L3 |
| L1 / L2 发现权限模型变化 | 升级 L3 |
| L1 / L2 发现生产配置变化 | 升级 L3 |
| 任意等级发现 Secret 泄露 | 高风险阻断，户部 + 兵部 + 刑部介入 |
| 任意等级发现无法回滚 | A 级确认 |
| 任意等级发现需求方向变化 | A 级确认 |
| 任意等级发现技术栈变化 | A 级确认 |

升级后必须：

```text
暂停执行；
史官记录升级原因；
尚书省重新调度；
必要时提交门下省审核；
涉及 A 级事项时等待皇帝确认。
```

## 数据边界自动升级

执行中发现以下情况必须升级：

```text
发现 Secret 明文 → 高风险阻断；
发现 .env 被提交 → 高风险阻断；
发现生产数据进入测试 / 日志 / 文档 → L3 或阻断；
发现需要新增数据库表 → L3；
发现需要新增向量索引来源 → L3；
发现工具输出含敏感信息 → 暂停并脱敏；
发现 artifact 包含敏感数据 → 停止发布并交由兵部 / 户部处理。
```
