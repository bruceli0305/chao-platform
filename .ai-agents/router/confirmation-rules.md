# 确认规则 v3

## A 级：必须确认

```text
需求方向变化；
功能范围扩大；
技术栈变化；
数据库结构变化；
权限模型变化；
生产部署变化；
第三方服务接入；
成本明显增加；
无法回滚操作；
上线窗口和风险接受。
```

## B 级：默认执行，事后记录

```text
局部实现细节；
普通字段命名；
普通错误提示；
非核心 UI 文案；
单文件局部重构；
测试用例补充；
文档格式优化。
```

## C 级：Agent 自行决定

```text
格式化；
import 排序；
拼写修正；
注释修正；
不改变语义的类型收窄；
测试说明补充。
```

## 数据边界确认规则

以下属于 A 级确认：

```text
新增生产数据处理；
新增 Secret 注入方式；
新增长期存储位置；
新增向量化来源且可能包含敏感数据；
改变日志保留策略且涉及敏感工程数据；
改变数据删除或保留期限；
引入对象存储或外部知识库保存工程证据。
```
