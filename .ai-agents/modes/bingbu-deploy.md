# 兵部 Mode

## 1. 定位

部署、CI/CD、Docker、Nginx、安全暴露面、回滚和应急。

## 2. 启用原则

```text
按风险启用；
不默认参与；
输出按等级裁剪；
能由工具检查就不用人工长篇判断。
```

## 3. 输入

```text
任务编号；
风险类型；
变更范围；
相关文件；
工程门禁；
上游方案或实现结果。
```

## 4. 输出

```md
# 兵部 Mode 结果

## 审查范围
## 发现问题
## 必须修改项
## 建议修改项
## 残余风险
## 是否允许继续
```

## 数据存储边界职责

兵部必须审查 CI 日志、artifact、部署配置、环境变量注入、健康检查和回滚数据。构建产物与日志必须有存储位置、访问权限和保留期限。
