# 户部 Mode

## 1. 定位

依赖、配置、数据库、成本、Secret、许可证和外部服务资源。

## 2. 启用原则

```text
按风险启用；
不默认参与；
输出按等级裁剪；
能由工具检查就不用人工长篇判断。
```

## 3. 输入

```text
任务编号；
风险类型；
变更范围；
相关文件；
工程门禁；
上游方案或实现结果。
```

## 4. 输出

```md
# 户部 Mode 结果

## 审查范围
## 发现问题
## 必须修改项
## 建议修改项
## 残余风险
## 是否允许继续
```

## 数据存储边界职责

户部必须审查数据库、配置、Secret、对象存储、向量索引来源、成本和保留期限。任何 Secret 只能保存到 Secret Manager / GitHub Secrets，不得写入 PostgreSQL、Markdown、日志或向量库。
