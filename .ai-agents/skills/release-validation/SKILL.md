# Skill: release-validation

## 1. 适用场景

发布前检查、构建产物、健康检查和回滚确认。

## 2. 不适用场景

```text
涉及 A 级确认但未确认；
超出当前任务范围；
需要新增未审查依赖；
涉及生产配置但未启用兵部；
涉及数据迁移但未启用户部和门下省。
```

## 3. 输入材料

```text
任务编号；
原始需求；
任务等级；
风险类型；
允许修改范围；
相关文件；
验收标准；
工程门禁。
```

## 4. 执行步骤

```text
1. 复述任务目标；
2. 检查范围和风险；
3. 读取必要上下文；
4. 执行最小必要动作；
5. 运行必要验证；
6. 输出结果、未覆盖内容和残余风险；
7. 通知史官记录关键事实。
```

## 5. 输出模板

```md
# Skill Result: release-validation

## 目标
## 执行内容
## 修改文件
## 验证结果
## 未覆盖内容
## 残余风险
## 是否建议交付
```

## 数据边界

本 Skill 必须声明输入和输出是否会进入 PostgreSQL、Markdown、pgvector、GitHub、Artifact Store 或 Workspace。禁止处理或持久化 Secret 明文、生产数据、个人隐私和完整工具长日志。
